CIty Bike

Rails, JavaScript, Jquery, Ajax, Redis, PostgreSQL, Git, Bootstrap, Heroku, Rspec

With the growth of cycling in the USA, it is vital to have an online website exclusively for bicycles, including professional ones. This Web application was built with Ruby on Rails as part of coursework. Implemented Actiocable from Rails 5 to show live updates on certain sections of the app. Used Devise and CanCanCan to manage user access. Ran security tests against the app.

The biggest challenge was to have the maximum security features to guarantee a secure personal information and payment to customers. The process to structure the page was based on the use of wireframes and drawings. Then I went to use draw.io to structure the database, especially the connections between the models, which was a challenge due to the complexity of the database.

After all the work, the result was great. With the use of RSpec I simulate the creation and update of multiple users, products and orders ensuring the excellent functioning of the page in different environments.

Ars reclama

Rails, JavaScript, Jquery, Ajax, PostgreSQL, Git, Bootstrap, Heroku, sass, zeplin, draw.io

Based on the fact doctors and physical therapists needs a better way to store and manipulate their patients information we create ars reclama. This app was made in collaboration with Francis Pujols, a UI Animator expert. We used zeplin as the collaboration tool to share the designs in code format.

As I was developing the application from scratch covering all software areas, many challenges were arising. The ability to modify the customer information in the database from outside the update form which is the normal way to do it, was a challenge and I solved it using a gem for this specific type of case.

Some of the challenges I can mention:

  • Print and save the customer information at the same time.
  • Modify the customer info from the show view.
  • Updating the information when modifying the customer info in the show view.

Most of them were solved using gems and new techniques.

The next step was to find the fastest way to access the data in the database to increase the speed of the web app. For this I used ajax and I put a limit on the amount of request that the page normally does in the index view. I reduced the amount of pages to load replacing them with modals.

Finally, the result was amazing. A web application that allow doctors and physical therapists to store and print their customer information. The response of our customers was positive we made their work a lot easier, some of them had no internet connection so we used docker to convert the app offline.

Integral

wordpress, css, zeplin

The integral company asked for a website where they can share their work, the service that they provide and multiple posts. WordPress was the fast and easy alternative. Another reason was that they already know how to use the post section and make basic configurations, So, they were looking for a website with this technology specifically.

The integral website requirements demanded the use of multiple plugins like elementor that is one of the best Page Builder. The big challenge for me was to learn all the plugins in a very short period, as it was my first time using it and the client needed it as soon as possible.

Some of the plugins used in this website:

  • Elementor pro
  • oceanwp widgets
  • ultimatefaq
  • essentialgrid

After all the work, I performed several tests on different mobile devices using an online platform to make sure the page is fully responsive. The final customer response was great, they loved our work.
